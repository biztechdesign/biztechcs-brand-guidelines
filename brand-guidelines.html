  • Do not recolor outside approved palette variations

Section 12

Imagery & Illustration Style

Photography Style

  • Subject: Real professionals in modern office environments; diverse, global teams
  • Mood: Bright, clean, optimistic — no dark or moody treatments
  • Color treatment: Natural, slightly warm-shifted; avoid over-saturated images
  • Composition: Subject centered or rule-of-thirds; avoid busy backgrounds
  • People: Candid over posed; authentic collaboration scenes preferred

Illustration Style

  • Type: Flat vector with subtle depth (isometric or 2.5D for complex scenes)
  • Colors: Drawn from brand palette — navy, blue, yellow, teal accents
  • Characters: Simple, geometric, diverse, gender-neutral where possible
  • Complexity: Medium detail — more than basic flat icons, less than realistic 3D

Imagery Do's and Don'ts

✓ Do
  • Use consistent color grading across sections
  • Crop to highlight the focal point
  • Use proper alt text on all images
  • Maintain consistent aspect ratios within a component
  • Use WebP format with JPG fallback
✕ Don't
  • Use clip art or generic stock photography
  • Mix photography and cartoon styles in the same section
  • Use watermarked or low-res images
  • Use images where text competes with background without a scrim
  • Use placeholder device frame mockups

Interaction & Motion Guidelines

Core Principles

  • Purposeful — Every animation communicates state or guides attention; no decorative motion
  • Subtle — Transforms are small (4px lift, not 20px); durations are short
  • Consistent — Same easing and duration across equivalent interactions

Standard Timing Reference

InteractionDurationEasing
Button hover200msease
Link color150msease
Card hover lift250msease
Dropdown open200msease-out
Modal open250msease
Accordion expand300msease-in-out
Scroll reveal600mscubic-bezier(0.16,1,0.3,1)
Stat counter2000msease-out

Animation Code Patterns

CSS
/* Card hover lift */
.card { transition: transform 0.25s ease, box-shadow 0.25s ease; }
.card:hover { transform: translateY(-4px); box-shadow: var(--shadow-lg); }

/* Arrow slide (Explore More →) */
.btn-text .arrow { transition: transform 0.2s ease; }
.btn-text:hover .arrow { transform: translateX(4px); }

/* Scroll reveal */
.reveal { opacity: 0; transform: translateY(24px); transition: opacity 0.6s cubic-bezier(0.16,1,0.3,1), transform 0.6s cubic-bezier(0.16,1,0.3,1); }
.reveal.visible { opacity: 1; transform: translateY(0); }

/* Reduced motion */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

Accessibility Guidelines

Color Contrast (WCAG 2.1 AA)

Text typeMinimum ratio
Normal text (< 18px)4.5:1
Large text (≥ 18px bold / ≥ 24px)3:1
UI components & icons3:1
✓ Passes: White on #1A4F9C blue · #1E2532 on white (AAA) · #F5A623 yellow on #0B1F3A navy
✕ Fails: Yellow #F5A623 text on white background — never use this combination.

Focus States

CSS
/* Never remove outline without a replacement */
:focus-visible {
  outline: 3px solid #1A4F9C;
  outline-offset: 2px;
  border-radius: 2px;
}
/* On dark backgrounds */
.dark :focus-visible { outline-color: #F5A623; }

ARIA Patterns

HTML
<!-- Navigation -->
<nav aria-label="Main navigation">

<!-- Mobile menu toggle -->
<button aria-expanded="false" aria-controls="mobile-menu">

<!-- Accordion -->
<button aria-expanded="false" aria-controls="panel-1">

<!-- Alert -->
<div role="alert" aria-live="polite">

<!-- Loading spinner -->
<div role="status" aria-label="Loading...">

<!-- Form error -->
<input aria-describedby="email-error" aria-invalid="true">
<span id="email-error" role="alert">Invalid email address</span>

Key Accessibility Rules

  • Minimum body font size: 14px — never smaller for flowing copy
  • Maximum line length: 70ch for paragraph text
  • All <img> must have alt text; decorative images use alt=""
  • All buttons need visible text or aria-label
  • Links must make sense out of context — avoid "click here" as standalone
  • Use <fieldset> + <legend> for radio/checkbox groups
  • Skip-to-content link at page top (visually hidden until focused)

Content Style Guide

Brand Voice Attributes

AttributeDescriptionExample
ConfidentOwn your expertise"We deliver." not "We try to deliver."
Outcome-firstLead with what the client gains"Scale faster" not "We offer cloud services"
CredibleBack claims with data"98% of clients stay — here's why"
HumanAvoid cold corporate-speak"Let's talk" not "Initiate a consultation"

CTA Wording Reference

ContextPreferred Wording
Primary page CTA"Talk to Experts", "Get Started", "Book a Free Call"
Service exploration"Explore [Service Name]", "See How It Works"
Case study"Read the Full Case Study"
Blog"Read Article"
Form submit"Request Consultation", "Send Message"
Hire section"Hire [Role]", "Get a Quote"
Avoid: "Submit", "Click here", "Learn More" (generic without context), "See More" without specificity

Grammar & Style Rules

  • Sentence case for headings and subheadings (not Title Case for everything)
  • Use Oxford comma in lists
  • Use active voice — "We built a platform" not "A platform was built by us"
  • Spell out one–nine; use numerals for 10+
  • Use "+" for stat callouts: "19+", "1,300+" — not "more than 19"
  • American English: "optimize" not "optimise"
  • Spell out acronyms on first use: "Customer Relationship Management (CRM)"
  • No exclamation marks in body copy — only acceptable in friendly UI microcopy

Content Do's and Don'ts

✓ Do
  • Start with the reader's problem or goal
  • Use short sentences for clarity
  • Include specific numbers and proof points
  • End sections with a clear next step
  • Use real client names and metrics in case studies
✕ Don't
  • Use buzzword chains: "synergistic end-to-end agile solutions"
  • Over-qualify: "potentially," "might possibly"
  • Start too many consecutive sentences with "We"
  • Lead with company history instead of client benefit
  • End pages without a CTA

Section 19

Do's and Don'ts

Visual

✓ Do
  • Use the defined color palette exclusively
  • Maintain clear H1 > H2 > H3 hierarchy
  • Use white space generously
  • Alternate section backgrounds (white / gray / navy)
  • Use device mockups with real screenshots
✕ Don't
  • Introduce off-brand colors without approval
  • Stack similar-weight headings together
  • Cram too much content into one section
  • Use the same background for every section
  • Mix icon sizes inconsistently in a component

Typography

✓ Do
  • Use Poppins for headings, Inter for body
  • Use the defined type scale
  • Weight 600–700 for headings
  • Line-height 1.7 for body paragraphs
  • Cap paragraph width at 70ch
✕ Don't
  • Mix 3+ font families on a page
  • Use arbitrary font sizes outside the scale
  • Set body copy in weight 700
  • Use line-height 1.0–1.2 for body text
  • Allow full-width paragraphs on wide screens

Color

✓ Do
  • Use #F5A623 yellow for primary CTAs
  • Use navy #0B1F3A for footer and dark heroes
  • Use brand blue for links and interactive elements
  • Test all text/background contrast pairs
  • Use semantic colors (green=success, red=error)
✕ Don't
  • Use yellow as a background for large sections
  • Use navy for body text on white
  • Use brand blue for purely decorative backgrounds
  • Assume yellow on white passes contrast (it doesn't)
  • Use brand blue for error states

Buttons & CTAs

✓ Do
  • One primary CTA per viewport section
  • Write action-oriented CTA text
  • Match button style to background context
  • Consistent padding and border-radius across buttons
✕ Don't
  • Put two yellow primary buttons side by side
  • Use "Submit", "Click Here", "Learn More" (generic)
  • Use yellow button on yellow/warm section background
  • Mix different border-radius values across buttons
